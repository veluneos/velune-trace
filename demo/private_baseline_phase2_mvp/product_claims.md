# Supported Product Claims

## Supported Now

VeluneOS can locally:

- load verified robotics Core Report Bundles
- create a private Reference set
- preserve an immutable initial Baseline Revision
- verify pinned Reference manifests
- load one explicit Target
- check compatibility against every Reference
- compare the Target against every Reference
- aggregate observed field differences
- produce reproducible JSON and Markdown reports

## Conservative External Statement

> VeluneOS locally compares a new robotics Target execution against
> a user-selected private Reference set and produces reproducible
> reports describing where observed evidence differences appeared.

## Unsupported Claims

VeluneOS must not currently be presented as automatically determining:

- root cause
- fault or liability
- safety
- severity
- normality
- regression
- improvement
- superiority
- production readiness
- regulatory compliance

## Interpretation Rule

A changed field is an observed difference.

It is not automatically a defect, anomaly, incident, or regression.
